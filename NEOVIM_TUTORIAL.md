# Neovim 新手教學

## 什麼是 Neovim？

Neovim 是 Vim 的現代化分支（fork），一個純鍵盤操作的文字編輯器。
跟一般編輯器最大的不同：你幾乎不需要滑鼠，靠鍵盤指令組合就能快速移動、修改、搜尋文字。

它跟 Vim 相容，但底層架構更現代，支援非同步插件、內建 LSP（程式碼智慧提示）、Lua 設定語言，社群插件生態也更活躍。

---

## 為什麼要學 Neovim？

### 問題：滑鼠編輯器的極限

一般編輯器（VS Code、Sublime 等）已經很好用，但重度打字/改 code 時，手要一直離開鍵盤去點滑鼠、拖曳選取、翻頁捲動，其實很浪費時間。

### 解決方案：Neovim 的三大優勢

**1. 純鍵盤操作，速度極快**
移動游標、刪除一段文字、複製貼上、跨檔案搜尋替換，全部靠鍵盤組合鍵完成，手不用離開主鍵盤區。熟練後編輯速度會明顯比滑鼠編輯器快。

**2. 幾乎在任何地方都能用**
SSH 連進遠端伺服器、Docker container、沒有 GUI 的環境，Neovim 都能直接跑在終端機裡。這對需要常常 SSH 進伺服器改設定檔的人特別有用。

**3. 搭配 Tmux 與 Claude Code 是完整的終端機工作流**
如果你已經在用 [Tmux 分割畫面工作](./TMUX_TUTORIAL.md)，Neovim 可以直接嵌進其中一個 pane，變成「全終端機」的開發環境：左邊 Claude Code、右邊 Neovim 改檔案，不用切換到其他視窗。

## 核心概念：Modal Editing（模式編輯）

Neovim 跟一般編輯器最大的差異：它有「模式」的概念。同一個按鍵在不同模式下做不同的事。

| 模式 | 用途 | 進入方式 |
|------|------|---------|
| **Normal**（一般模式） | 移動游標、下指令（刪除、複製…） | 預設模式 / 按 `Esc` |
| **Insert**（插入模式） | 打字輸入文字 | `i`、`a`、`o` 等 |
| **Visual**（視覺模式） | 選取文字範圍 | `v`、`V`、`Ctrl+v` |
| **Command**（命令模式） | 執行指令（存檔、離開、搜尋替換…） | `:` |

> 最重要的觀念：**你大部分時間應該待在 Normal 模式**，只在真正要打字時才進 Insert 模式，打完立刻按 `Esc` 回到 Normal。新手最常見的錯誤是待在 Insert 模式不出來，導致完全用不到 Neovim 的威力。

## 安裝

```sh
# macOS（使用 Homebrew）
brew install neovim

# 確認安裝成功
nvim --version
```

啟動方式：

```sh
nvim 檔案名稱      # 開啟指定檔案
nvim              # 開啟空白編輯器
nvim .            # 開啟目前目錄（檔案總管）
```

## 循序漸進的上手路徑

不要想一次學完所有指令，照這個順序練習，每一步都紮實再往下：

### 第一步：能存活（開檔、打字、存檔、離開）

```
i        進入 Insert 模式（開始打字）
Esc      回到 Normal 模式
:w       存檔（write）
:q       離開（quit）
:wq      存檔並離開
:q!      強制離開（不存檔）
```

> 光是這 6 個指令，你就能正常編輯並離開 Neovim 了。先熟悉這步再往下。

### 第二步：移動游標（不用方向鍵）

```
h j k l   左 下 上 右（取代方向鍵）
w         跳到下一個單字開頭
b         跳到上一個單字開頭
0         跳到行首
$         跳到行尾
gg        跳到檔案開頭
G         跳到檔案結尾
:數字     跳到指定行（例如 :42 跳到第42行）
```

### 第三步：基本修改指令

```
x         刪除游標所在字元
dd        刪除整行（並複製起來）
yy        複製整行（yank）
p         貼上（paste，在游標後方）
u         復原（undo）
Ctrl+r    重做（redo）
```

### 第四步：組合技（Neovim 的精髓）

Neovim 指令可以組合成「動詞 + 範圍」的句子，這是效率的來源：

```
dw        刪除到下一個單字開頭（delete word）
d$        刪除到行尾
d0        刪除到行首
ciw       刪除游標所在單字並進入 Insert（change inner word）
dd        刪除整行
3dd       刪除 3 行
d3w       刪除 3 個單字
```

> 規則：`d`（delete）/ `c`（change）/ `y`（yank）+ 移動指令 = 對那段範圍做動作。學會這個組合邏輯後，其他指令都能自己推理出來。

### 第五步：搜尋與取代

```
/關鍵字        往下搜尋
?關鍵字        往上搜尋
n              跳到下一個符合處
N              跳到上一個符合處
:%s/舊/新/g    全檔案取代（substitute）
:%s/舊/新/gc   全檔案取代，每次替換前詢問確認
```

## 常用指令總表

### 移動

| 指令 | 說明 |
|------|------|
| `h` `j` `k` `l` | 左 下 上 右 |
| `w` / `b` | 下一個 / 上一個單字 |
| `e` | 跳到單字結尾 |
| `0` / `$` | 行首 / 行尾 |
| `gg` / `G` | 檔案開頭 / 結尾 |
| `Ctrl+d` / `Ctrl+u` | 往下 / 往上翻半頁 |
| `%` | 跳到對應的括號 `()` `{}` `[]` |

### 編輯

| 指令 | 說明 |
|------|------|
| `i` / `a` | 在游標前 / 後插入 |
| `I` / `A` | 在行首 / 行尾插入 |
| `o` / `O` | 在下方 / 上方新增一行並插入 |
| `x` | 刪除單一字元 |
| `dd` / `yy` | 刪除 / 複製整行 |
| `p` / `P` | 貼在游標後 / 前 |
| `u` / `Ctrl+r` | 復原 / 重做 |
| `.` | 重複上一個編輯動作（非常好用） |
| `>>` / `<<` | 整行向右 / 向左縮排 |

### 視覺模式（選取）

| 指令 | 說明 |
|------|------|
| `v` | 逐字選取 |
| `V` | 整行選取 |
| `Ctrl+v` | 區塊選取（直向選取多行同一欄位） |
| 選取後 `d` | 刪除選取範圍 |
| 選取後 `y` | 複製選取範圍 |
| 選取後 `>` / `<` | 整段縮排 / 反縮排 |

### 檔案 / 視窗

| 指令 | 說明 |
|------|------|
| `:w` | 存檔 |
| `:q` / `:q!` | 離開 / 強制離開 |
| `:wq` / `ZZ` | 存檔並離開 |
| `:e 檔名` | 開啟另一個檔案 |
| `:sp 檔名` | 上下分割視窗開啟檔案 |
| `:vsp 檔名` | 左右分割視窗開啟檔案 |
| `Ctrl+w` + 方向鍵 | 在分割視窗間切換 |

## 常見問題

**Q：我按了一堆鍵，畫面變得很奇怪？**
先按 `Esc` 多次回到 Normal 模式，這是最安全的「重置」動作。90% 的困惑都是因為卡在 Insert 或其他模式。

**Q：怎麼離開 Neovim？**
`Esc` 回 Normal 模式，再輸入 `:q` 離開（有改動要先 `:w` 存檔，或用 `:wq` 一次完成，或 `:q!` 放棄改動強制離開）。

**Q：忘記某個指令在幹嘛？**
在 Normal 模式輸入 `:help 指令名稱`，例如 `:help dd`。內建說明文件非常完整。

**Q：改壞了想全部復原？**
一直按 `u` 復原到滿意為止；如果整個檔案都想放棄，用 `:q!` 不存檔離開。

## 下一步

熟悉以上基礎、能流暢編輯檔案之後，可以進一步研究：

- `:help` 內建說明系統 - 每個指令都有詳細文件
- [vimtutor](https://neovim.io/doc/) - 終端機輸入 `vimtutor` 就有互動式教學
- 設定檔 `~/.config/nvim/init.lua` - 用 Lua 客製化按鍵與外觀
- 插件管理工具（如 `lazy.nvim`）- 加上檔案總管、模糊搜尋、LSP 自動補全
- `vim-tmux-navigator` - 讓 Tmux pane 與 Neovim 視窗之間用同一組快捷鍵切換，詳見 [Tmux 教學](./TMUX_TUTORIAL.md)
